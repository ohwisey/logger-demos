My supplement stack is done. Now plug it into my Vitality dashboard from Episode 0.

First find my dashboard: it is my own copy of vitality-base. If the folder is
not on this computer, ask me for the GitHub link to my copy and clone it here
first, then continue.

Make a dashboard copy of my stack as ONE self-contained HTML file:
1. Everything inline, no outside requests. It runs sealed inside a dashboard
   tile, so anything not in the file will not load.
2. No localStorage in this copy. The dashboard provides a bridge: save with
   await window.Vitality.save(data) and read it back with
   await window.Vitality.load(). You never write the bridge yourself.
3. If window.Vitality is missing (the file opened on its own), fall back to
   saving on the device, so the same file works both places.
4. Same theme, same features. Do not dumb anything down.

Save it as public/tiles/fuel.html inside the dashboard folder, commit, and
push. Then tell me to wait a minute for the deploy and refresh my dashboard:
the Fuel tile is now my supplement stack.
