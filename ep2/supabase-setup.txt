FREE SUPABASE SETUP (basic saving for your logger)

This is the bare-bones version. It just lets your own logger SAVE what you log,
synced to a free Supabase project. It is intentionally minimal. The clean
architecture, row-level security, and the production-ready backend are the paid
version. This is enough to get your saves working.

====================================================================

1. Make a free Supabase project
   - Go to supabase.com and sign up (free).
   - Click New project. Give it any name. Pick a password. Wait for it to spin up.

2. Create a minimal table
   - In your project, open the SQL editor (left sidebar).
   - Paste the block below and run it. It makes one basic workout_log table.

-- Minimal workout_log table: just enough to save a lift and its sets.
create table if not exists workout_log (
  id         bigint generated by default as identity primary key,
  lift       text not null,
  sets       jsonb not null default '[]',   -- [{ weight, reps, done }]
  created_at timestamptz not null default now()
);

3. Grab your project keys
   - Open Settings, then API.
   - Copy the Project URL and the anon public key. Your logger uses these two
     to talk to Supabase.

4. Save and read
   - To save a lift, insert a row:
       insert into workout_log (lift, sets) values ('Bench press', '[]');
   - To read your lifts back:
       select * from workout_log order by created_at;

That is it. Your logger can now save. When you want proper multi-user security
and the clean architecture, that is the paid episode.
